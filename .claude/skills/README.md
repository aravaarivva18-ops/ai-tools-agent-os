# Скилы этого харнесса

First-party скилы, которые едут вместе с шаблоном — самодостаточны, ничего внешнего
ставить не нужно. Каждый покрывает одну стадию workflow и ссылается на соседей через
`[[имя]]`.

| Скил | Когда срабатывает |
|------|-------------------|
| `spec-loop`        | Мастер-навигатор по всему циклу: explore → propose → plan → auto → gates → archive. Точка входа, если неясно, с чего начать. |
| `plan-tasks`       | Написание `tasks.md`: декомпозиция на маленькие проверяемые шаги. |
| `verify-loop`      | Перед коммитом каждой задачи: 2 слоя верификации (ревью + гейты). |
| `north-star`       | Инварианты и защита от дрейфа между change'ами. |
| `brownfield`       | Работа на СУЩЕСТВУЮЩЕМ коде: фича / рефактор / эпик. Контекст из кода, инварианты = что нельзя сломать. `start.sh --brownfield`. |
| `interactive-flow` | Пошаговый старт с утверждением: гейт + чек-лист документов после каждого этапа, без «OK» дальше не идёт. `start.sh --interactive`. |

Поток: **spec-loop** оркеструет всё; **plan-tasks** — на этапе плана; **verify-loop** и
**north-star** — на каждой итерации авто-блока. **brownfield** и **interactive-flow** —
два входных режима (greenfield-из-PRD против из-кода; схлопнутый против пошагового),
композятся друг с другом.

## Внешние зависимости (НЕ вендорятся, ставит получатель)

Эти вещи не лежат в шаблоне сознательно — они устанавливаются отдельно, каждая своим
способом. Шаблон работает и без опциональных, просто с меньшим числом слоёв.

- **Команды OpenSpec** (`/opsx:explore`, `/opsx:propose`, `/opsx:apply`, `/opsx:archive`) —
  это НЕ скилы, а слэш-команды, которые генерит `openspec init` под выбранные инструменты.
  Обязательны для workflow. Ставится: `npm install -g @fission-ai/openspec && openspec init`
  (именно `@fission-ai/openspec` — голый `openspec` в npm это чужая заглушка v0.0.0 без бинаря).
- **`codex` CLI** (опционально) — нужен для кросс-модельного ревью в `loop.sh`
  (`codex exec` ревьюит diff второй моделью). Если не установлен — ревью падает на
  claude-`verifier` subagent, цикл продолжает работать. Ставится отдельно (codex-плагин).
- **`fallow`** (рекомендуется для TS/JS) — детерминированный анализатор
  [github.com/fallow-rs/fallow](https://github.com/fallow-rs/fallow); ревьюер ВСЕГДА его
  гоняет (`scripts/fallow-review.sh`) и учитывает в отчёте (dead code, дубли, сложность,
  границы, зависимости, PR-risk). Нет инструмента / не TS-JS — скрипт пишет пометку и не
  падает. Ставится: `npm i -D fallow`  или  `cargo install fallow-cli`.
- **`gitleaks`** (опционально) — секрет-скан в Gate 2 `gates.sh`. Нет инструмента —
  гейт просто пропускается (best-effort).

## Добавляешь свой скил?

Клади в `.claude/skills/<имя>/SKILL.md` с фронтматтером `name` + `description`
(триггеро-ориентированное описание «когда использовать»). Ссылайся на существующие
через `[[имя]]`. Держи скил коротким и про «когда/как», а не «что» — «что» уже в
`AGENTS.md`/`PROMPT.md`.
