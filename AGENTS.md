## 📐 Регламенты сессии
- **Solo Loop v10 + Compaction**: Субагенты запрещены. Статус сессии считывается из `implementation_plan.md` через `PlanningWithFiles`.
- **Авто-инжект стандартов (Agent OS)**: При запуске `PlanningWithFiles` автоматически генерируется файл `vault/standards.md` на основе используемых технологий (Python, Frontend, Marketing). Агент ОБЯЗАН прочитать этот файл при инициализации и строго следовать стандартам.
- **Слэш-команды ИИ (Mental Commands)**:
  - `/discover-standards` — извлечь найденные в коде паттерны качества и сохранить в `/Users/rus/ai-tools/standards/`.
  - `/inject-standards` — принудительно обновить файл `vault/standards.md` в текущем проекте.
- **CLI-команды `agy`**:
  - `agy run` — запуск авто-лечения ошибок через `test_healer.py`.
  - `agy search "<query>"` — семантический поиск по хандоффам.
  - `agy init` — инициализация конфига проекта.
  - `agy test` — инкрементальный запуск pytest по измененным файлам (git diff).
- **Karpathy & levelsio**: Линейный плоский код (макс 2-3 уровня абстракции), SQLite по умолчанию.
- **JIT Testing**: Использовать `agy test` для быстрого запуска связанных тестов по измененным файлам. pytest запускать только перед шипингом.
- **Commit-on-DoD**: Коммиты делать только после закрытия логической вехи (DoD) и зеленых тестов.
- **Внутренний состязательный цикл (Adversarial Loop)**: Для нетривиальных задач (КП, ТЗ, стратегии, код >30 строк) агент обязан сгенерировать черновик (Draft) и жестко раскритиковать его (проверка на воду, ИИ-мусор Stop-Slop, оверхед) в размышлении (thinking) перед выводом финального ответа.
- **Форматирование ссылок**: Ссылки на файлы и символы оформлять строго через [name](file:///path/to/file) без бэктиков вокруг имени (запрещено использовать бэктики в `[name](file://...)`).
- **Дизайн-системы Open Design**: При верстке UI и генерации стилей агент обязан проверить наличие готовой дизайн-системы бренда в папке [vault/design-systems/](file:///Users/rus/ai-tools/vault/design-systems/) и использовать её цвета, шрифты и CSS-токены из соответствующего `DESIGN.md` для обеспечения премиального нешаблонного визуального стиля.
- **Размер модулей (LoC)**: Предпочитать добавление новых файлов/модулей вместо раздувания существующих. Длина отдельного файла не должна превышать 500 строк кода (исключая тесты). Если файл приближается к 800 строкам, принудительно выносить новую логику в отдельные модули.
- **Самодокументируемый API (Keyword-Only)**: Запрещено использовать неявные позиционные параметры булевого, числового или опционального типа в сигнатуре новых функций/методов. Все такие параметры должны быть жестко объявлены как keyword-only (после знака `*`, например, `def process(*, force: bool):`), чтобы вызовы выглядели как `process(force=True)` вместо `process(True)`.
- **Запрет на хелперы-однодневки**: Запрещено создавать мелкие вспомогательные методы или функции, которые вызываются в коде ровно один раз. Инлайнить их по месту использования (YAGNI).
- **Минимальные изменения (Minimal Change Principle)**: Вносить строго те изменения в код, которые необходимы для выполнения ТЗ. Категорически запрещено рефакторить соседний код, форматировать файлы или вносить "косметические" правки, не относящиеся к задаче, чтобы минимизировать размер diff и исключить регрессионные ошибки.
- **Obsidian**: Логировать хандоффы в Daily Note через `session_logger.py`.
- **Синхронизация понимания (Problem Restate)**: При получении любой нетривиальной задачи (>3 шагов или создание новых файлов) агент обязан переформулировать ТЗ своими словами в виде Target State (Ожидаемое конечное состояние), зафиксировать его в `implementation_plan.md` и получить одобрение пользователя (например, "ок", "погнали") перед написанием любого рабочего кода.
- **Использование agent-runtime**: Для простых задач генерации текста (one-shot) использовать прямой вызов SDK. Для сложных многошаговых агентов, требующих вызова инструментов (tool use) и циклов выполнения (LLM loops), использовать установленный пакет `agent-runtime` вместо написания кастомных циклов.
