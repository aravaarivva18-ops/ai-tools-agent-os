# OSINT Repository Analysis: agentmemory (rohitg00)
**Repository URL:** https://github.com/rohitg00/agentmemory

## 🎯 Назначение и Механика
`agentmemory` предлагает легковесное иерархическое хранилище памяти для ИИ-агентов.
- **Механика**: Использует плоскую структуру Markdown-заметок с автоматической перелинковкой и легкий SQLite-индекс для поиска контекста прошлых сессий. Вместо тяжелых облачных векторных БД, вся память хранится локально в воркспейсе, гарантируя скорость чтения <10мс.

## 📊 Критический анализ YAGNI & Solo Loop
- **Плюсы**: Идеально совместим с офлайн-режимом работы и принципом экономии памяти хоста. Никаких сторонних API-зависимостей и докеров.
- **Минусы (Bloat)**: Включает встроенные коннекторы к 10+ фреймворкам (CrewAI, Autogen), которые нам не нужны.
- **Вердикт**: **Адаптировать концептуально**. Наша утилита `tools/llm_wiki.py` и база Obsidian (`vault/`) реализуют аналогичный паттерн ("LLM Wiki вместо RAG"). Использовать опыт для оптимизации линтинга связей заметок.
