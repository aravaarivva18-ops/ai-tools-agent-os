# OSINT Repository Analysis: context-mode (mksglu)
**Repository URL:** https://github.com/mksglu/context-mode

## 🎯 Назначение и Механика
`context-mode` виртуализирует контекстное окно агента, подменяя сырой вывод тяжелых инструментов (чтение файлов, git log, списки директорий) на легкие проиндексированные записи в локальной базе данных SQLite.
- **Механика**: Вместо передачи всего файла объемом >1000 строк в чат, файл индексируется в SQLite, а LLM получает краткую выжимку или поисковый индекс. Агент может запрашивать конкретные строки по мере необходимости через SQL-запросы, экономя до 98% токенов контекста.

## 📊 Критический анализ YAGNI & Solo Loop
- **Плюсы**: Идеальное решение для работы с легаси-кодовыми базами огромного объема. Защищает контекст от засорения длинными трассировками.
- **Минусы (Bloat)**: Высокая сложность интеграции, требующая переопределения стандартных инструментов ACI.
- **Вердикт**: **Адаптировать концептуально**. Мы уже используем правила ACI (чтение файлов строго диапазонами до 100 строк, grep-фильтрация). Внедрить жесткое ограничение на чтение больших дампов в Solo Loop v10.
