# 🔍 Анализ Agno (Phidata)
**Категория:** AI Agent Framework / SDK  
**GitHub:** [agno-agi/agno](https://github.com/agno-agi/agno)  

---

## 🏗️ 1. Архитектурный разбор
Agno основан на концепции **AgentOS** (операционная система агентов), которая разделяет объявление агентов от рантайма выполнения.
*   **Декларативное описание:** Агенты объявляются на чистом Python с явным указанием `model`, `instructions`, `tools`, `memory` и `knowledge`.
*   **Отсутствие Graph/Chain оверхеда:** В отличие от LangChain, Agno не пытается упаковать все в цепочки вызовов, отдавая приоритет обычному императивному Python-коду.
*   **Интегрированный UI & База данных:** Включает из коробки PgVector-интеграцию для Knowledge Base и собственный локальный веб-интерфейс для отладки.

---

## ⚖️ 2. Соответствие принципам Antigravity v3.2
*   **Solo Loop Compliance:** Agno поддерживает команды (teams) из нескольких агентов, но базово ориентирован на качественную работу одиночного агента (Solo Agent), предоставляя ему богатый набор инструментов.
*   **YAGNI (Избыточность):** Agno тянет за собой собственный сервер выполнения, FastAPI-обертки и интеграцию с базами данных. Для локального монорепозитория `@ai-tools/` это лишний оверхед (bloatware).
*   **Performance (<2s):** Инициализация Agno-агента происходит быстро, но запуск всего стека AgentOS (FastAPI + UI + DB connections) выходит за рамки 2 секунд.

---

## 💡 3. Что адаптировать в @ai-tools/
1.  **Декларативное описание инструментов:** Идея легкой обертки над Python-функциями для превращения их в ИИ-инструменты с автогенерацией JSON-схемы (это то, что мы применили в нашем RAG и Obsidian CLI).
2.  **Дисциплина баз знаний (Knowledge):** Организация базы знаний как автономного модуля, отделенного от системного промпта.
