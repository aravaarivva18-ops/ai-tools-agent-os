## 🏗️ 1. Архитектурный разбор
`browser-use` — это Python-библиотека для автономного взаимодействия агентов с браузером.
*   **Playwright Engine:** Управляет реальным инстансом браузера (Chromium, Firefox).
*   **Screenshot & DOM Parsing:** На каждом шаге агент делает скриншот страницы, считывает дерево DOM и отправляет это в LLM для принятия решения.
*   **Rust Core (Beta):** В версии v0.13+ критические секции рендеринга и работы с DOM переписаны на Rust для увеличения скорости.
*   **No Hardcoded Selectors:** Агент кликает и заполняет формы, ориентируясь на визуальную разметку и текст, а не на хрупкие XPath/CSS-селекторы.

---

## ⚖️ 2. Соответствие принципам Antigravity v3.2
*   **Solo Loop Compliance:** Полностью совместим. Позволяет одному агенту совершать действия во внешней веб-среде.
*   **Latency & Performance (Нарушено):** Пошаговый цикл (скриншот $\rightarrow$ LLM $\rightarrow$ действие $\rightarrow$ рендеринг) занимает десятки секунд на каждый шаг. Для утилит со скоростью старта < 2с это неприемлемо.
*   **TDD (Нарушено):** Практически невозможно написать быстрые герметичные офлайн-тесты (<100мс) для веб-агента, так как требуется запуск браузера и реальных сайтов.

## 💡 3. Что адаптировать в @ai-tools/
1.  **Инструмент для сторонней автоматизации:** Саму библиотеку `browser-use` не стоит включать в ядро системы. Однако концепцию адаптивного считывания DOM и скриншотов полезно использовать в качестве специализированного внешнего скрипта для парсинга сайтов на заказ.
2.  **Дисциплина `uv`:** Проект жестко рекомендует использование менеджера `uv` для изоляции окружения, что мы уже внедрили и используем.
