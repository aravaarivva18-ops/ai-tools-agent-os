# DOX — Contract: Bitrix Knowledge Base & RAG Index

Локальный регламент разработки для модуля базы знаний по 1С-Битрикс.

## Архитектура и YAGNI
1. **База данных:** Использовать SQLite для хранения текстов документов, метаданных и векторов в виде JSON/BLOB. Запрещено добавлять сторонние векторные СУБД (Chroma, Milvus) без прямой инструкции.
2. **Эмбеддинги:** В продакшене использовать Gemini API (`models/text-embedding-004`). В тестовом/офлайн-режиме использовать детерминированное векторное хэширование слов, чтобы исключить внешние сетевые запросы.
3. **Обновление:** Использовать инкрементальное обновление документов на основе SHA256-хэшей контента.

## Требования к тестированию
1. Все тесты RAG-чатботов должны быть герметичными. Сетевые вызовы `urllib.request.urlopen` или API-запросы к Gemini должны быть полностью замоканы.
2. Запрещено выполнять реальные обращения к хостам в юнит-тестах.
