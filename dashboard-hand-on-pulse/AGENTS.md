# DOX — Dashboard: Hand on Pulse (Рука на пульсе)

Streamlit дашборд коммерческого класса для клиента "Таргет Медиа", агрегирующий результаты SEO, рекламы (Яндекс.Директ) и YouTube-генерации.

## 📁 Структура проекта
- `streamlit_app.py` — Главная панель Streamlit с интерактивной визуализацией метрик (CPL, ROI, spend, leads) и наложением событий changelog на графики.
- `main.py` — API сервер на FastAPI для интеграции внешних сервисов.
- `models.py` — Схемы данных SQLAlchemy (клиенты, проекты, лиды, changelog).
- `sync.py` — Логика импорта и синхронизации данных (Google Sheets, Yandex Direct, Excel).
- `run_all.py` — Единый скрипт запуска (API + Streamlit).
- `db.py` — Настройки пула соединений SQLite/PostgreSQL.
- `init_db.py` — Скрипт инициализации схемы БД.

## 📐 Регламенты разработки
1. **Отклик и Скорость:** Отрисовка Streamlit-графиков Plotly должна укладываться строго в $< 2.0\text{s}$ (алгоритмическая сложность запросов и кэширования $O(N+M)$).
2. **Логгирование событий:** Все вызовы бизнес-моделей (SEO, YouTube) логируют изменения в базу `dashboard.db` с помощью `tools/dashboard_logger.py`.
3. **Безопасность:** Секреты API и учетные данные OAuth Яндекса загружаются строго из `.env` файла. Конкатенация строк в SQL строго запрещена (только prepared statements).
4. **Тесты:** Вся бизнес-логика синхронизации и API должна быть покрыта тестами TDD (позитивными и негативными) в файлах `test_api.py` и `test_sync.py`.
