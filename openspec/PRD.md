# PRD — Поиск и реализация улучшений для ai-tools

## Продукт
Аналитическая сессия и инструменты для поиска архитектурных, производительных и стоимостных улучшений в кодовой базе `ai-tools`. 

## Для кого / пользователи
Для пользователя (разработчика/предпринимателя) с целью повышения надежности, скорости работы CLI и снижения затрат на токены API при работе ИИ-агента.

## Проблема
По мере развития системы в кодовую базу могут проникать лишние импорты, неоптимальные алгоритмы обработки контекста (например, неточный подсчет токенов), дублирование логики поиска и избыточные RAG-запросы, что замедляет работу CLI и увеличивает расходы на API.

## Цели
1. Провести полный аудит импортов и времени старта CLI.
2. Выявить узкие места в работе с токенами и контекстом в `SoloLoopV10`.
3. Разработать архитектурные решения по оптимизации (например, RAG Search Cache).
4. Оформить результаты в виде конкретных задач-кандидатов (changes) в OpenSpec.

## Не-цели (вне скоупа продукта)
- Написание новых крупных функций (например, графического интерфейса), не связанных с оптимизацией текущего ядра.
- Переход на другие языки программирования.

## Ключевые требования / фичи
- **Анализ импортов**: Проверка всех модулей на "тяжелые" импорты и перевод их на ленивую (lazy) загрузку для сохранения времени импорта CLI < 20 мс.
- **Оптимизация токенов**: Проектирование точного подсчета токенов в `compact_context` (замена грубой оценки по словам на точный токенизатор).
- **Кэширование RAG**: Проектирование локального кэша для `semantic_search.py`, чтобы исключить повторные запросы к векторной базе.

## Ограничения
- Сохранение полной совместимости с Python 3.14 на macOS.
- 100% прохождение существующего тест-сьюта (278 тестов).
- Строгое соблюдение YAGNI и YAGNI-аудита (без оверхеда).
