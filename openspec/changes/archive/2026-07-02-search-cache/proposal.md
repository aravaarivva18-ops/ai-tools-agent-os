# Предложение: Кэширование RAG-поиска для ускорения сессий

## Проблема
При каждом запуске сессии или обращении к планировщику RAG-поиск инициирует загрузку векторной модели `FastEmbed` и повторный расчет близости векторов. Это создает задержку в 1.5-2 секунды, замедляя отзывчивость системы.

## Решение
Реализовать локальный файловый кэш `vault/search_cache.json` для сохранения результатов RAG-поиска по тексту запроса:
1. Кэш считывает `mtime` (время изменения) папки `vault/handoffs`.
2. Если в кэше есть запись для запроса и ее `mtime` совпадает с текущим `mtime` папки `vault/handoffs`, дайджест отдается из кэша мгновенно без загрузки тяжелых библиотек.
3. При изменении/добавлении файлов хандоффов кэш автоматически инвалидируется и перестраивается.

## Ожидаемый эффект
- Снижение времени выполнения RAG-поиска с ~2 секунд до < 5 миллисекунд для повторяющихся запросов.
- Исключение ненужных накладных расходов на инициализацию моделей эмбеддингов.
