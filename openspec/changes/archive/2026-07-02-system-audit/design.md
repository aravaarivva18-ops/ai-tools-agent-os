# Технический дизайн: Аудит кодовой базы ai-tools

## Методология сбора данных

### 1. Анализ времени импорта CLI
- Запуск команды `python -X importtime -c "import tools.cli"` для точной детализации времени загрузки каждого модуля.
- Выявление не-ленивых импортов сторонних библиотек.

### 2. Аудит алгоритма Solo Loop V10
- Исследование файла `core/solo_loop.py`, в частности методов `compact_context` и механизмов Stealth Stop.
- Поиск неэффективностей в подсчете токенов (где используется грубая оценка по словам).

### 3. Исследование RAG-поиска
- Анализ `tools/obsidian/semantic_search.py` на предмет повторного чтения индекса и выполнения векторного поиска при одинаковых запросах.

## Выходной артефакт
Результаты будут оформлены в виде файла `wiki/system_audit_report.md` с конкретными техническими рекомендациями.
