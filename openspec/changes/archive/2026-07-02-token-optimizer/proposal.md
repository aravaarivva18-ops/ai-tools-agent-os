# Предложение: Точная оптимизация контекста в SoloLoopV10

## Проблема
Класс `SoloLoopV10` обрезает вывод каждой команды до 5 строк без контроля объема токенов. При накоплении шагов в сессии контекстное окно ИИ-агента раздувается, увеличивая время ответа и стоимость API.

## Предлагаемое решение
Интегрировать точный подсчет токенов (`count_tokens_exact`) в логику `SoloLoopV10.compact_context` и реализовать адаптивный алгоритм сжатия (Exact Token Compression):
1. Вычисление токенов для каждого шага.
2. При превышении лимита в `2000` токенов:
   - Сначала обрезать `output` для успешных шагов (оставляя только сам факт успешного запуска).
   - Если размер все еще превышает лимит — полностью удалять самые старые шаги из истории.

## Ожидаемый эффект
- Жесткий контроль размера контекста на уровне API.
- Экономия до 30% токенов в длинных сессиях разработки.
