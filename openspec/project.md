# Технический стек и конвенции ai-tools

## Стек технологий
- **Язык**: Python >= 3.12 (основная рабочая среда Python 3.14 на macOS).
- **Менеджер пакетов**: `uv` (управление workspace members).
- **Линтинг и форматирование**: `ruff check`, `ruff format`.
- **Проверка типов**: `mypy` (с конфигом в `pyproject.toml`).
- **Тестирование**: `pytest` (с плагином `pytest-socket` для изоляции сети).
- **База знаний**: Obsidian (REST API через `obsidian_cli.py` + плоские Markdown-файлы).

## Структура проекта
- `tools/` — ядро системы (скрипты автоматизации, CLI, логирование).
- `tools/tests/` — юнит-тесты ядра.
- `vault/` — база знаний Obsidian (Daily Notes, справочники, findings).
- `wiki/` — черновики правил и системные спецификации.

## Архитектурные правила
- **YAGNI**: Никакого оверхеда и "хлама". Три строки простого кода лучше, чем сложная абстракция.
- **Комментарии**: Не писать комментарии, объясняющие *что* делает код. Писать только *почему* (неочевидные ограничения).
- **Ограничение файлов**: Максимум 500 строк (редко до 800) на файл. Большие файлы дробить.
- **Изоляция сети**: Все тесты запускаются строго с флагом `--disable-socket`. Сетевые вызовы в тестах мокируются.
