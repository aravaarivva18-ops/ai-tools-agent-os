# Стандарты Фронтенда и Чистого Кода

Этот навык описывает жесткие стандарты верстки, оптимизации производительности и архитектуры компонентов, обязательные для выполнения при проектировании любых веб-интерфейсов в воркспейсе.

---

## 📱 1. Верстка и Адаптивность (Mobile-First)

- **Mobile-First подход:** Стилизация начинается с мобильных экранов. Префиксы адаптивности (например, `sm:`, `md:`, `lg:` в Tailwind CSS) используются только для масштабирования интерфейса вверх, а не вниз.
- **Обязательные брейкпоинты для тестов:** **320px** (мобильные), **768px** (планшеты), **1024px** (ноутбуки), **1440px+** (десктоп).
- **Размер областей клика:** Все интерактивные элементы (кнопки, ссылки) на мобильных экранах должны иметь размер области клика не менее **44x44px** для предотвращения ложных нажатий.
- **Предотвращение сдвигов (CLS):** Все изображения `<img>` обязаны иметь жестко заданные атрибуты `width` и `height` (или CSS-свойство `aspect-ratio`), чтобы браузер заранее резервировал под них место на странице.
- **Ленивая загрузка (Lazy Loading):** Применяйте `loading="lazy"` ко всем изображениям, находящимся ниже первого экрана (below-the-fold).

---

## ♿ 2. Доступность интерфейсов (WCAG 2.1 AA)

- **Семантический HTML:** Вместо навешивания обработчиков клика `onClick` на `div` или `span`, всегда используйте нативные теги `<button>` (для действий) и `<a>` (для переходов).
- **Фокус клавиатуры:** Все интерактивные элементы должны быть доступны по клавише `Tab` и иметь четко видимое состояние фокуса (запрещено использовать `outline: none` без предоставления кастомных стилей фокуса).
- **Атрибуты ARIA:**
  * Каждое изображение должно иметь атрибут `alt` (для декоративных элементов используйте пустой `alt=""` или `aria-hidden="true"`).
  * Кнопки без текста (например, кнопки-иконки закрытия модальных окон) обязаны иметь `aria-label="Закрыть окно"`.
- **Контрастность:** Контраст обычного текста должен быть не менее **4.5:1** по отношению к фону, а для крупных шрифтов и графических элементов — не менее **3:1**.

---

## 🚀 3. Оптимизация производительности (Core Web Vitals)

- **Largest Contentful Paint (LCP):** Время отрисовки основного контента должно быть **≤ 2.5 секунд**. Самое важное изображение на первом экране (LCP-элемент) должно иметь атрибут `fetchpriority="high"` и загружаться без lazy loading.
- **Interaction to Next Paint (INP):** Время отклика интерфейса на клик пользователя должно быть **≤ 200 мс**. Избегайте блокирования основного потока (main thread) тяжелыми JS-вычислениями дольше 50 мс.
- **Размер бандла:** Ограничивайте размер начального бандла (Initial Bundle) до **200 КБ** в сжатом виде (gzipped). Используйте динамический импорт (`import()`) для разделения тяжелого функционала на части.

---

## 📐 4. Чистая Архитектура Компонентов

- **Разделение логики и представления:** Выносите логику запросов к API и стейта в кастомные хуки, оставляя компоненты максимально «глупыми» (presentational).
- **Размер файлов:** Если файл компонента превышает **200 строк кода**, разбейте его на несколько более мелких, узкоспециализированных компонентов.
- **Колокация ресурсов (Colocation):** Файлы тестов, стилей, типов и сам код компонента должны находиться в одной папке (например, `src/components/Button/`).
- **Запрет на "Lorem Ipsum":** Никогда не используйте шаблонный текст. Применяйте реальные текстовые данные, чтобы заранее увидеть возможные переносы строк, переполнения блоков и проблемы верстки.
