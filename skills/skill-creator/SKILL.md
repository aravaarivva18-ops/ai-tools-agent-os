# Skill Creator & Standardizer

Этот навык определяет правила и предоставляет шаблоны для создания стандартизованных JIT-навыков (Just-In-Time Skills) в монорепозитории `@ai-tools/`.

## 🏗️ Структура нового навыка
Каждый новый навык обязан создаваться в отдельной директории `skills/<имя-навыка>/` со следующей структурой:
```
skills/<имя-навыка>/
├── SKILL.md             # Инструкции для ИИ-агента с YAML-метаданными
├── scripts/             # Дополнительные скрипты автоматизации (опционально)
├── examples/            # Примеры входных/выходных файлов и использования (опционально)
└── tests/               # Быстрые оффлайн-тесты (<100мс) для скриптов навыка (опционально)
```

## 📐 Правила проектирования и Метаданные (v3.2)
1. **YAML Frontmatter**: Каждый `SKILL.md` обязан начинаться с метаданных:
   ```yaml
   ---
   name: уникальное-имя-навыка
   description: Краткое бизнес-описание назначения навыка.
   ---
   ```
2. **Типобезопасность выходов (Type-Safe Outputs)**:
   - Все результаты работы скриптов навыка должны валидироваться с помощью Pydantic-моделей из [tools/tool_validator.py](file:///Users/rus/ai-tools/tools/tool_validator.py).
   - Запрещено возвращать сырые неструктурированные словари или строки без предварительного прохождения через `validate_llm_output`.
3. **Декларативные схемы (Agno-style)**:
   - Функции навыка, предназначенные для вызова ИИ, должны автоматически генерировать спецификации аргументов с помощью `generate_tool_schema` для предотвращения ошибок рассогласования контрактов.
4. **YAGNI & Ограничение абстракции**:
   - Логика навыка должна иметь максимум 2 уровня абстракции.
   - Запрещено использовать сложные внешние агентные фреймворки типа LangChain/Agno напрямую. Наследуйте только простые функции.

## 🔄 Чек-лист верификации
- [ ] Файл `skills/<имя-навыка>/SKILL.md` содержит валидный YAML-блок с `name` и `description`.
- [ ] Описание аргументов функций задокументировано в docstrings (Google/Sphinx/list style).
- [ ] Выходные структуры описаны как Pydantic `BaseModel`.
- [ ] Реализованы оффлайн-тесты для скриптов навыка в папке `tests/` с покрытием основных ветвей.
