# Глобальный стандарт: Разработка Frontend и UI/UX (HTML / CSS / JS)

Этот стандарт обязателен для выполнения всеми ИИ-агентами при верстке страниц, создании интерфейсов и написании клиентского кода.

---

## 🎨 1. Типографика и Эстетика (Anti-Generic)
Избегайте скучных стандартных шрифтов и дешёвого «ИИ-вида».
*   **Шрифтовые пары:** Для заголовков используйте акцидентный шрифт **`Cabinet Grotesk`** (или *Sora*). Для основного текста — геометрический **`Satoshi`** (или *Inter*).
*   **Контрастность:** Тексты должны быть читаемыми. Перед сдачей UI обязательно валидируйте контрастность по стандарту WCAG через линтер:
    `npx @google/design.md lint DESIGN.md`

---

## 🌀 2. Тактильный отклик и Анимация (Spring Physics)
Интерфейс должен ощущаться «живым» и физически отзывчивым при взаимодействии.
*   **Тактильный клик:** Все кнопки (`button`, `.btn`, `[role="button"]`) при клике должны совершать упругое сжатие с легким размытием.
    ```css
    button:active, .btn:active {
        transform: scale(var(--spring-snappy-scale, 0.96)) translateY(1px);
        filter: var(--spring-snappy-blur, blur(0.5px));
        transition: transform 0.15s cubic-bezier(0.175, 0.885, 0.32, 1.275);
    }
    ```
*   **Плавные переходы:** Вместо линейных переходов `transition: all 0.3s` используйте точечные переходы для конкретных свойств с использованием пружинных безье-кривых.

---

## ✍️ 3. Чистый JavaScript / TypeScript (YAGNI & Clean Code)
Пишите код без избыточных абстракций, ориентируясь на простоту поддержки.

### 🚫 Запрет оператора `else`
Избегайте ветвления `else` для уменьшения вложенности кода. Всегда используйте паттерн **«Ранний возврат» (Early Return)** или тернарные операторы.

*   *Плохо:*
    ```javascript
    function checkAccess(user) {
        if (user.isAdmin) {
            renderDashboard();
        } else {
            showError();
        }
    }
    ```
*   *Хорошо:*
    ```javascript
    function checkAccess(user) {
        if (!user.isAdmin) {
            return showError();
        }
        renderDashboard();
    }
    ```

### 📦 Чистые импорты
Избегайте импорта всех сущностей (`import * as ...`) или избыточных псевдонимов (`import { foo as bar }`), если это не продиктовано конфигурацией фреймворка. Используйте только явный именованный импорт.

### ✂️ Минимизация функций (levelsio YAGNI)
Держите логику в рамках одной функции, пока не возникнет реальная необходимость повторного использования кода в других местах. Не создавайте хелперы и утилиты «про запас». Сокращайте число переменных за счет инлайнинга выражений однократного использования.
