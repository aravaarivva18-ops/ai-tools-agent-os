# Implementation Plan: Dashboard MVP Audit & Refactoring

Этот документ описывает план аудита, YAGNI-рефакторинга и улучшения архитектуры аналитического дашборда «Рука на пульсе» (Target Media).

## 🏗️ 1. Контекст и Проблематика
Дашборд сквозной аналитики и плановых KPI объединяет данные из API Яндекс.Директ/Метрики и Google Таблиц. Текущая кодовая база содержит дублирующие файлы прототипов, неиспользуемые скрипты и имела ошибки путей импорта при запуске тестов. Задача — навести архитектурный порядок (YAGNI), исправить тесты и гарантировать готовность к переносу БД на PostgreSQL.

---

## 🎯 2. Критерии готовности (Definition of Done - DoD)
- [ ] Все неиспользуемые и дублирующие файлы прототипов удалены.
- [ ] Рабочий Streamlit-фронтенд корректно переименован в `streamlit_app.py`, а старый мок-файл удален.
- [ ] Скрипт автозапуска `run_all.py` перенастроен на правильные файлы.
- [ ] Все 100+ автотестов проекта (включая тесты дашборда) проходят успешно через `make test` и `uv run pytest`.
- [ ] Модели данных (`models.py`) и подключение к БД (`db.py`) поддерживают DateTime с таймзоной для бесшовного перехода на PostgreSQL (`timestamptz`).
- [ ] Секреты (JWT, OAuth) вынесены в конфигурацию `config.py` с чтением из `.env` и не имеют жесткого кодирования.
- [ ] Запущен цикл самообучения агента `self_improve.py` в конце сессии.

---

## 📋 3. Список задач (Task List) и Приоритеты (MoSCoW)

### 📌 Must Have (Критически важные задачи)
1. **Task 1: Исправление импортов в тестах (Completed)**
   - *Описание*: Решить проблему `ModuleNotFoundError: No module named 'dashboard_mvp'` при запуске тестов.
   - *Решение*: В `pyproject.toml` в секцию `tool.pytest.ini_options` добавлен путь поиска `./tools`. Тесты успешно находят модуль.

2. **Task 2: YAGNI Очистка и Переименование Фронтендов (Completed)**
   - *Описание*: Устранить путаницу между `app.py` и `streamlit_app.py`.
   - *Решение*: Актуальный код из `app.py` (с интеграцией API) перенесен в `streamlit_app.py` (перезаписав мок-версию). Файл `app.py` удален.
   - *Описание*: Удалить неиспользуемый файл-коннектор `etl_pipeline.py`.
   - *Решение*: Файл удален. Логика синхронизации полностью инкапсулирована в `sync.py`.

3. **Task 3: Перенастройка оркестратора запуска (Completed)**
   - *Описание*: Обновить `run_all.py` для корректного запуска нового фронтенда `streamlit_app.py` вместо удаленного `app.py`.
   - *Решение*: Пути запуска скорректированы, uvicorn и streamlit запускаются синхронно.

4. **Task 4: Избавление от Deprecated-кода по времени (Completed)**
   - *Описание*: Заменить deprecated-метод `datetime.utcnow()` на timezone-aware `datetime.now(timezone.utc)` в `yandex_oauth.py` и `security_utils.py`.
   - *Решение*: Вызовы заменены, предупреждения линтера устранены.

5. **Task 5: Готовность к PostgreSQL (Supabase/Neon) (Completed)**
   - *Описание*: Обновить типы данных в `models.py` для совместимости со стандартом Конституции (`timestamptz`).
   - *Решение*: Поле `expires_at` в `Integration` переведено на `Column(DateTime(timezone=True))`. Совместимость с SQLite сохранена благодаря динамическому движку `db.py`.

### 📌 Should Have (Важные задачи)
6. **Task 6: Проверка Безопасности (Секреты)**
   - *Описание*: Аудит `main.py` и `sync.py` на наличие захардкоженных API токенов Яндекса или Google Sheets.
   - *Верификация*: Проверка конфигов, перенос дефолтных значений в `.env.example`.

### 📌 Could Have (Желательные задачи)
7. **Task 7: Финализация и сбор метрик**
   - *Описание*: Запуск `python3 tools/self_improve.py` для фиксации результатов рефакторинга и обновления Obsidian-базы знаний.
