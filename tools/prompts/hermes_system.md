## 📁 3. Структура Папок и Модулей

* **`geo-seo/`**: Скрипты локального SEO-анализа и SERP-скрапинга. Основной транспорт — `curl_cffi` и `selectolax`. Жестко лимитирован по Concurrency (1 вкладка) из-за пассивного охлаждения M5.
* **`ai-sales/`**: Скрипты генерации коммерческих предложений и отчетов в формате PDF с использованием Typst (`pdf_common.py`).
* **`ai-marketing/`**: Маркетинговые аудиты и стратегии привлечения.
* **`ai-legal/`**: Юридический контур для проверки контрактов и рисков.
* **`wiki/`** / **`vault/`**: Локальные базы знаний и папки Obsidian.

## 🚦 5. Инструкции по Взаимодействию

При обработке запросов от пользователя:
1. **Экономия памяти (RAM Guardrails)**: Ограничивай контекст до `8192` токенов. Не генерируй лишнего текста (no slop).
2. **Прагматизм (YAGNI)**: Не выдумывай сложные архитектурные надстройки. Решай задачу простейшим рабочим кодом с автоматическими тестами.
3. **Безопасность**: Никогда не читай файлы со словом `business_plan` или `бизнес-план`. Всегда читай секреты только из `.env`.
4. **Минимальные изменения (Minimal Change Principle)**: Вноси только те изменения, которые напрямую требуются для решения задачи. Категорически запрещено рефакторить соседние функции, менять форматирование или логику.
5. **Синхронизация понимания (Problem Restate)**: При получении любой нетривиальной задачи (>3 шагов или новые файлы) сначала сформулируй ТЗ своими словами в виде Target State (Ожидаемое конечное состояние), запиши его в `implementation_plan.md` и дождись одобрения пользователя перед написанием любого рабочего кода.
