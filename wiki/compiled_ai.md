# Compiled AI (Скомпилированный ИИ)

Методология проектирования ИИ-систем, при которой LLM вызывается один раз на этапе компиляции/генерации для создания детерминированного программного кода (артефакта), а выполнение транзакций идет без обращения к ИИ (zero-token execution).

## 📍 Когда предлагать этот метод (Критерии выбора)

Следует активно рекомендовать архитектуру Compiled AI, если задача соответствует следующим критериям:
1. **Высокая частота транзакций** (минимизирует расходы на токены).
2. **Требование к детерминизму** (отсутствие Stochasticity/Entropy).
3. **Жесткие ограничения по времени отклика (Latency)** (мгновенное локальное исполнение).
4. **Наличие четких спецификаций или правил** (логика ветвления, регулярные выражения, формулы).
5. **Требование к безопасности и аудиту** (весь путь принятия решений должен логироваться в виде статического кода, исключая prompt injection в продакшене).

## 🛠️ Структура реализации
* **Спецификация (YAML/JSON):** Описание условий и логики.
* **Генерация бизнес-логики (LLM):** Одноразовый вызов модели для сборки скрипта.
* **Валидация:**
  * Безопасность (Semgrep / Bandit / линтеры)
  * Синтаксис (ruff, mypy)
  * Песочница (Execution tests)
* **Запуск:** Статический код в детерминированном рантайме.
* **Гибридный вариант (Code Factory):** Зажатый в строгую схему вызов LLM (Safety Sandwich) только для семантических подзадач извлечения, где чистый код бессилен.
