В этом файле накапливаются кандидаты на правила, сгенерированные автоматически движком `task_observer.py` на основе анализа сессий.
После апрува они переносятся в основные файлы правил (AGENTS.md или справочники в vault/reference/rules/).

### 💡 Предложенное правило: Безопасный парсинг Git Diff
*   **Правило**: При анализе git diff для проверки конфигурационных файлов (TOML, YAML, JSON) парсер обязан игнорировать строки метаданных diff и hunk-заголовки (начинающиеся с `@@`, `---`, `+++`, `diff`, `index`). Также необходимо использовать валидацию синтаксиса элементов (проверку кавычек для правил линтера) и отслеживать границы блоков по закрывающим скобкам (`]`, `}`), чтобы избежать ложных срабатываний на несвязанные свойства.
*   **Почему**: Устранение ложных срабатываний в `linter_config_guard.py` при безопасных изменениях типа `line-length`.
*   **Промпт**: `Всегда игнорируй hunk headers (строки с @@) и метаданные diff при парсинге конфигураций во избежание ложных срабатываний.`

### 💡 Предложенное правило: Фильтрация деструктивных CLI команд
*   **Правило**: При создании регулярных выражений для блокировки опасных CLI-команд (например, `git push --force`) всегда используйте негативный просмотр вперед (negative lookahead), например `(?!-with-lease)`, чтобы избежать ложной блокировки безопасных флагов, содержащих опасное слово как подстроку.
*   **Почему**: Устранение ложного блокирования безопасного флага `git push --force-with-lease`.
*   **Промпт**: `При фильтрации опасных CLI команд проверяй, не является ли слово частью безопасного флага с помощью negative lookahead (?!-with-lease).`

### 💡 Предложенное правило: Лимит попыток авто-лечения (Iteration Budget)
*   **Правило**: При автоматическом исправлении ошибок/тестов ИИ-агент имеет право сделать не более 5 последовательных попыток наложения патчей и запуска тестов. Если после 5-й попытки тесты всё ещё падают, агент обязан прекратить автоматические попытки, восстановить исходное состояние (откатить патч) и предоставить пользователю подробный отчет о причинах неудачи с вариантами решений.
*   **Почему**: Предотвращение зацикливания модели и неконтролируемого списания токенов (защита баланса API).
*   **Промпт**: `Соблюдай лимит в 5 попыток при исправлении тестов. Если баг не решен за 5 итераций, остановись, откати изменения и напиши отчет.`
